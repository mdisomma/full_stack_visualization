Purpose

This dashboard allows users to visualize trends in D.C. 311 data; 311 is a non-emergency phone number that people can call in many cities to find information about services, make complaints, or report problems. This data (which includes complaint types, locations, requisite agencies, dates, etc.) has been provided by the D.C. OpenData project, and has been tracked since 2009. For the sake of efficiency, we have restricted these visualizations to 2019 data (which still boasts about 300,000 records worth of information); interact with the maps/charts and see if you can spot any interesting trends!

Analysis

For our analysis, we wanted to study the effects of some of the variables we created/derived from the original 311 dataset on response time. What helps decide variations in response to a request? Is it the month it is called in? The DC ward it's called in from? The agency that responds to it? Or something else? For this test, we decided to run a multiple regression with variables (mostly dummies) chosen from a decision tree. Because the original data were obviously not in a "dummy" format, we used Python to create them and then scored them using a Random Forest. The primary variables we created dummies out of were:

  • Complaint Type
  • Weekend/Weekday
  • Ward
  • Month of Year

    • Unfortunately, none of the variables were nearly as explanatory as we had hoped. The subsequent regression was also not very explanatory, and the residuals seem patterned around consistent underprediction.
